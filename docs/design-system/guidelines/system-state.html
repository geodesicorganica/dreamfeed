Signal → meaning → behavior → flash rate
SignalMeaningBehaviorFlash rate
AMBER Active process · warning · queue delay Persists while condition is live ≤ 2 Hz (observe)
RED Halted · failure · blockage Holds until acknowledged; demands intervention 3–5 Hz (critical)
GREEN Build success · handoff · LIVE state Transient on action success; persistent on record & LIVE none
CYAN Selection · link · focus ring · info Persists while selected or focused; never an alert none
NEUTRAL Nominal — compiling, routing, syncing Silent. No color, no notification none
Alert interlock — Flashing stops immediately when cursor enters or keyboard focus moves into the affected block. The persistent text state and trace remain available after motion stops. Every flashing alert must simultaneously render a non-flashing equivalent naming: severity · affected object · diagnostic condition · timestamp · next available action.
